# Personal Preferences

## Communication
- Español para mensajes, inglés para código/comentarios/commits

## Code Style
- Prefer functional programming over OOP
- Pure functions only (no side effects)
- Explicit parameters (no defaults)
- Strict typing everywhere

## Workflow
- Run tests after code changes
- Show diffs for changes, not full files
- Git: conventional commits (feat/fix/docs)

## Tools
- OS: Windows 11
- Terminal: PowerShell
- Editor: VS Code

## Token Efficiency
- Use `/btw` for questions that don't need context
- Read files only when explicitly needed
- Batch related edits in single operation
- Use `/compact` before hitting 50% context
